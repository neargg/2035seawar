## 一、背景

### 1.1 现状问题

2035大海战 V5.5 已上线运行（nearapp.xyz），核心玩法完整（6阵营/135卡/17机制/3种对战模式），但存在两个核心体验短板：

1. **AI 毫无策略性**：`aiTurn()` 函数中所有候选动作通过 `Math.random()` 随机选择（第1951行），导致 AI 会做出"用8费航母攻击1血无人机被反死"等荒谬操作，人机模式体验极差，新玩家无法从中学习策略。

2. **战斗零视觉反馈**：`doAttack()` 函数直接修改数据后调用 `render()` 全量重绘（第1890行），所有变化瞬间完成。攻击唯一的反馈是音效 + 日志文字，玩家无法直观感知"谁打了谁、打了多少、谁死了"。

### 1.2 业务目标

- 提升人机模式留存率（新玩家首局体验 → 次留）
- 降低新手学习门槛（通过 AI 操作动画理解游戏机制）
- 为后续 P1 需求（设置面板/难度选择/新手引导）奠定基础

## 二、目标

| 目标 | 衡量指标 |
|------|----------|
| AI 具备策略性决策能力 | 困难档 AI 对中等玩家胜率 ≥ 45% |
| AI 体现阵营特色 | 不同阵营 AI 前5回合行为模式有可感知差异 |

| 动画不拖慢节奏 | 单次攻击动画链 ≤ 1.2s，回合切换 ≤ 0.8s |
| 性能可控 | 60fps 设备动画无卡顿，低端设备可降级 |

## 五、需求描述（EARS 原则）

### 5.1 AI 智能化升级

#### REQ-AI-01：难度选择界面（Event-driven）

> **When** 玩家在开始界面点击"人机对战"，**the system shall** 显示难度选择子菜单，包含"简单""普通""困难"三个选项，每个选项附带一句话描述。

难度描述：
- 简单：「AI 会犯一些错误，适合熟悉规则」
- 普通：「AI 会评估局势做出合理决策」
- 困难：「AI 会精心计算每一步，谨慎应战」

#### REQ-AI-02：评分制决策引擎（Ubiquitous）

> **The system shall** 为 AI 每个候选动作计算评分，按评分排序选择动作执行，而非随机选择。

评分公式：

```
score = baseScore + killBonus + hqBonus + tacticalBonus - riskPenalty
```

| 评分维度 | 条件 | 分值 |
|----------|------|------|
| 击杀目标 | 目标当前HP ≤ 攻击者ATK | +50 |
| 攻击HQ | 目标为敌方HQ | +30 |
| 斩首加成 | 目标为HQ且攻击者有 hqAtkBonus | +(15 + hqAtkBonus×5) |
| 低血目标 | 目标HP ≤ 3（非必杀） | +15 |
| 部署高费 | 部署费用 ≥ 5 | +(cost × 3) |
| 部署中费 | 部署费用 3-4 | +(cost × 2) |
| 部署低费 | 部署费用 ≤ 2 | +5 |
| 推进前线 | 支援线→前线推进 | +15 |
| 指令卡-过牌 | 抽牌类指令 | +12 |
| 指令卡-群伤 | 全体伤害类指令 | +18 |
| 指令卡-治疗 | 恢复类指令（HQ<50%时） | +20 |
| 指令卡-增益 | 增益类指令 | +10 |
| 指令卡-封锁 | 封锁/干扰类指令 | +14 |
| 母舰部署 | carrierSpawn 单位 | +10 额外 |
| 反击风险 | 攻击者会被反击且会死 | -40 |
| 反击损耗 | 攻击者会被反击但不会死 | -(反击伤害 × 3) |
| 攻击无效 | 攻击无法击杀且目标反伤高 | -15 |
| 部署浪费 | 手牌有更高价值卡但选了低的 | -5 |

#### REQ-AI-03：难度分档策略（State-driven）

> **While** AI 难度设置为「简单」，**the system shall** 从评分 Top 50% 的动作中随机选择（保留一定随机性）。

> **While** AI 难度设置为「普通」，**the system shall** 选择评分最高的动作。

> **While** AI 难度设置为「困难」，**the system shall** 选择评分最高的动作，并在攻击前模拟反击结果，优先选择能击杀且自身存活的动作；若场上无紧急威胁，优先部署高费单位建立场面优势。

三档差异汇总：

| 行为 | 简单 | 普通 | 困难 |
|------|------|------|------|
| 动作选择 | Top50%随机 | Top1 | Top1+前瞻 |
| 反击预判 | 不考虑 | 考虑（规避送死） | 考虑+优先击杀 |
| 资源管理 | 有K就花 | 留K给高费卡 | 囤K等关键卡 |
| 部署优先级 | 随机费用 | 按评分 | 高费优先+场面评估 |
| 指令卡使用 | 30%概率 | 按评分 | 按评分+时机判断 |

#### REQ-AI-04：阵营特性策略权重（Ubiquitous）

> **The system shall** 根据 AI 所使用的阵营，在评分基础上叠加阵营策略权重，使不同阵营 AI 展现不同行为风格。

| 阵营 | 策略倾向 | 权重调整 |
|------|----------|----------|
| 苍龙 | 中后期厚甲推进 | 潜艇部署+10，护盾/维修指令+8，前3回合倾向防守 |
| 利维坦 | 制空+航母展开 | 飞行器部署+12，航母部署+15，制空指令+10 |
| 北极熊 | 铁甲防守反击 | 高HP单位部署+10，防守指令+12，不急于推进 |
| 武士 | 快攻突袭 | 低费单位部署+8，前3回合推进+15，隐身单位+10 |
| 雄鹰 | 蜂群铺场+穿透斩首 | 无人机/蜂群部署+12，穿透导弹+15，EMP指令+10 |
| 灰狼 | 无人机海+过载 | TB2/Akinci部署+12，过载攻击+8，布雷+10 |

#### REQ-AI-05：AI 逐步操作动画（Event-driven）

> **When** AI 执行回合操作，**the system shall** 在每个动作之间插入 700ms 间隔（困难档 500ms），配合动画展示，使玩家能看清 AI 的每一步操作。

> **If** 动画开关被关闭，**then the system shall** 将动作间隔缩短为 200ms，仅保留日志反馈。

#### REQ-AI-06：AI 阵营随机化（Event-driven）

> **When** 开始一局人机对战，**the system shall** 为 AI 随机分配一个阵营（从6个阵营中随机选择），而非固定使用利维坦牌组。玩家阵营保持苍龙不变（V6.1再做阵营选择）。

### 5.2 战斗动画系统

#### REQ-AN-01：动画总控架构（Ubiquitous）

> **The system shall** 建立动画队列管理器（AnimationQueue），所有战斗动画按序执行，前一个动画完成后才触发下一个。

架构设计：

```
AnimationQueue {
  queue: []          // 待执行动画队列
  playing: false     // 是否正在播放
  enabled: true      // 动画开关

  add(animFn)        // 添加动画到队列
  playNext()         // 播放下一个
  skip()             // 跳过当前（动画关闭时）
  clear()            // 清空队列
}
```

> **If** 动画开关被关闭，**then the system shall** 跳过所有动画，直接更新状态并渲染。

#### REQ-AN-02：部署动画（Event-driven）

> **When** 单位从手牌部署到支援线，**the system shall** 播放部署动画：手牌卡牌缩小→飞向目标空格→放大定位，持续 400ms，同时播放部署音效。

#### REQ-AN-03：推进动画（Event-driven）

> **When** 支援线单位推进到前线，**the system shall** 播放推进动画：单位从支援线滑入前线对应空格，带轻微拖尾效果，持续 350ms。

#### REQ-AN-04：普通攻击动画（Event-driven）

> **When** 非导弹单位发起攻击，**the system shall** 播放冲刺动画：攻击者向目标方向位移 30%→回弹原位，持续 300ms，随后触发命中反馈。

#### REQ-AN-05：导弹攻击动画（Event-driven）

> **When** missile=true 的单位发起攻击，**the system shall** 播放导弹飞行动画：在 Canvas 层绘制导弹图标从攻击者位置沿弧线飞向目标，飞行 500ms，到达后触发爆炸粒子特效。

#### REQ-AN-06：命中反馈（Event-driven）

> **When** 任何攻击造成伤害，**the system shall** 在目标位置显示伤害飘字（红色数字向上飘浮并淡出，持续 600ms），同时目标卡牌红色闪烁 2 次。

> **When** 攻击触发反击，**the system shall** 在攻击者位置显示反击伤害飘字（橙色数字），同时攻击者卡牌橙色闪烁。

#### REQ-AN-07：单位摧毁动画（Event-driven）

> **When** 单位 HP 归零被摧毁，**the system shall** 播放摧毁动画：卡牌碎裂效果（scale + rotate + opacity 渐变）持续 400ms，同时在 Canvas 层播放爆炸粒子特效（8-12个粒子向外扩散）。

#### REQ-AN-08：HQ 受击动画（Event-driven）

> **When** HQ 受到伤害，**the system shall** 播放 HQ 震动动画（translate 抖动 3 次，持续 300ms），HQ 卡牌边框红色闪烁，同时战场区域轻微震动。

#### REQ-AN-09：指令卡生效动画（Event-driven）

> **When** 指令卡被打出，**the system shall** 播放指令生效动画：战场全屏短暂色调变化（对应阵营色，200ms），指令卡图标在屏幕中央浮现放大后淡出（400ms）。

#### REQ-AN-10：回合切换横幅（Event-driven）

> **When** 回合切换时，**the system shall** 在战场顶部滑入横幅提示"你的回合"/"敌方回合"，持续 800ms 后滑出。

#### REQ-AN-11：动画开关（State-driven + Optional）

> **Where** 设置面板中存在动画开关，**the system shall** 允许玩家开启/关闭战斗动画。

> **While** 动画开关为关闭状态，**the system shall** 跳过所有 CSS/Canvas 动画，所有状态变化即时渲染，AI 动作间隔缩短为 200ms。

#### REQ-AN-12：Canvas 特效层（Ubiquitous）

> **The system shall** 在战场区域上方覆盖一个透明 Canvas 元素（pointer-events: none），用于绘制粒子特效、导弹轨迹和爆炸效果，不影响 DOM 交互。

Canvas 规格：
- 尺寸：与 `.battle-area` 一致，resize 时同步
- z-index：高于卡牌 DOM，低于遮罩层
- 粒子系统：支持位置/速度/生命/颜色/大小 5 个属性
- 帧率：requestAnimationFrame，目标 60fps

## 六、流程说明

### 6.1 AI 回合执行流程（V6.0）

```
AI回合开始
  │
  ├─ startTurn('ai') → K恢复 + 抽牌 + 回合横幅动画
  │
  ├─ 等待 800ms（回合横幅动画时间）
  │
  ▼
┌─→ aiStep()  ←─────────────────┐
│   │                            │
│   ├─ buildActions()            │
│   │   生成所有合法候选动作      │
│   │                            │
│   ├─ scoreActions()             │
│   │   对每个动作评分            │
│   │   (含阵营权重)              │
│   │                            │
│   ├─ selectAction()             │
│   │   简单: Top50%随机          │
│   │   普通: Top1                │
│   │   困难: Top1+前瞻           │
│   │                            │
│   ├─ executeAction(act)         │
│   │   ├─ 部署 → 部署动画(400ms) │
│   │   ├─ 推进 → 推进动画(350ms) │
│   │   ├─ 攻击 → 攻击动画(800ms) │
│   │   └─ 指令 → 指令动画(500ms) │
│   │                            │
│   ├─ 等待 700ms（简单/普通）    │
│   │   等待 500ms（困难）        │
│   │                            │
│   ├─ 还有合法动作且K足够？─────┤
│   │     AND 安全计数 < 15      │
│   │                            │
│   ├─ 否 ──→ aiFinishTurn()     │
│   │          切换到玩家回合     │
└────────────────────────────────┘
```

### 6.2 攻击动画时序

```
doAttack() 被调用
  │
  ├─ 计算伤害值（ ATK / 电子干扰 / 穿透等）
  │
  ├─ AnimationQueue.add(攻击冲刺动画, 300ms)
  │     攻击者向目标位移30% → 回弹
  │
  ├─ AnimationQueue.add(命中反馈, 300ms)
  │     目标红色闪烁 + 伤害飘字
  │     [Canvas] 命中火花粒子
  │
  ├─ 计算反击伤害
  │
  ├─ if 有反击:
  │   AnimationQueue.add(反击动画, 300ms)
  │     目标小幅冲刺 + 反击飘字
  │
  ├─ if 目标HP≤0:
  │   AnimationQueue.add(摧毁动画, 400ms)
  │     卡牌碎裂 + [Canvas] 爆炸粒子
  │
  ├─ if 攻击者HP≤0:
  │   AnimationQueue.add(摧毁动画, 400ms)
  │
  ├─ if 目标是HQ:
  │   AnimationQueue.add(HQ震动, 300ms)
  │
  ├─ AnimationQueue.onComplete → render()
  │
  总时长: 普通攻击 ≈ 900ms
          导弹攻击 ≈ 1100ms
          含击杀 ≈ 1300ms
```

### 7.2 设置面板（新增）

在开始界面底部新增「⚙️ 设置」按钮，点击弹出设置面板：

| 设置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 战斗动画 | 开关 | 开 | 开启后播放所有战斗动画 |
| 音效 | 开关 | 开 | 开启后播放音效 |
| AI 操作速度 | 滑块 | 普通 | 慢(1000ms)/普通(700ms)/快(400ms) |

设置存储到 `localStorage`，刷新后保持。

### 7.3 动画与交互的配合

- 玩家操作时：动画是即时的（玩家点击→动画播放→状态更新），不阻塞操作
- AI 操作时：动画是顺序的（动作1→动画→间隔→动作2→动画→...），玩家在动画期间不可操作
- 联机模式：动画在收到 state 后播放，不影响操作权

## 十、边界场景与异常处理

| 场景 | 处理方式 |
|------|----------|
| AI 无合法动作 | 立即结束回合（aiFinishTurn） |
| AI 动作执行出错 | catch 异常，跳过该动作，继续下一个 |
| 动画队列堆积 | 队列最大长度 10，超出时跳过旧动画直接渲染最终状态 |
| Canvas 不可用（极旧浏览器） | 降级为纯 CSS 动画，跳过粒子特效 |
| 动画期间页面失焦 | requestAnimationFrame 自动暂停，恢复后继续 |
| AI 安全计数耗尽（15步） | 强制结束回合 |
| 联机模式 | 动画在客户端本地播放，不影响服务器状态同步 |

## 十一、技术实现要点

### 11.1 代码结构变更

V5.5 为 2300 行单文件。V6.0 在单文件内做模块化分区（不拆分文件，保持部署简单）：

```javascript
// ==================== 动画系统 ====================
class AnimationQueue { ... }
class ParticleSystem { ... }     // Canvas 粒子
class BattleAnimator { ... }     // 战斗动画封装

// ==================== AI 引擎 ====================
class AIEngine {
  constructor(difficulty, faction) { ... }
  buildActions() { ... }
  scoreAction(act) { ... }
  selectAction(actions) { ... }
  executeStep() { ... }          // 单步执行+动画+间隔
}

// ==================== 设置管理 ====================
class Settings {
  load() / save() / get() / set()
}
```

### 11.2 关键改动点

| 文件位置 | 改动 |
|----------|------|
| 开始界面 HTML | 新增难度选择子菜单 + 设置按钮 |
| `aiTurn()` | 重写为 `AIEngine` 类，评分制+逐步执行 |
| `doAttack()` | 拆分为 `calculateAttack()` + `playAttackAnimation()` |
| `render()` | 增加动画状态判断，动画期间不全量重绘 |
| `newGame()` | 增加 difficulty/aiFaction 参数 |
| 新增 | AnimationQueue / ParticleSystem / BattleAnimator / Settings 类 |
| 新增 | Canvas 元素覆盖战场区域 |
| CSS | 新增动画 keyframes + 飘字/闪烁/震动样式 |
| localStorage | 存储动画/音效/速度设置 |

### 11.3 性能考量

- Canvas 粒子数量上限：单次特效 ≤ 15 个粒子，同时存活 ≤ 30 个
- 动画用 CSS transform/opacity（GPU 加速），避免触发 layout
- `render()` 在动画期间做增量更新（只改变化的 DOM），避免全量 `innerHTML`
- AI 评分计算复杂度 O(n×m)，n=手牌数(≤9)，m=目标数(≤10)，单次 < 1ms
