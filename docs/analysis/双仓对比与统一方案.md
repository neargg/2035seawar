## 三、执行步骤

### 第 1 步：巴蒂合并（你来协调）

巴蒂需要在 Mac mini 上执行：

```bash
# 1. 克隆 GitHub 主仓
git clone https://github.com/neargg/2035seawar.git 2035seawar-github
cd 2035seawar-github

# 2. 添加 Gitee 作为第二个 remote，拉取巴蒂的最新代码
git remote add gitee https://gitee.com/nearzhao/2035seawar.git
git fetch gitee

# 3. 合并 Gitee 的 src/ 文件到 GitHub
#    把 Gitee 上的最新 V6 代码和服务端文件复制进来
git checkout gitee/main -- index-v6.html game-engine.js cards-data.js game-server.js nginx-2035sea.conf

# 4. 移动到正确目录
mkdir -p src server
mv index-v6.html src/
mv game-engine.js server/
mv cards-data.js server/
mv game-server.js server/
mv nginx-2035sea.conf deploy/

# 5. 合并 Gitee 上的文档和 WorkBuddy 分析
git checkout gitee/main -- CODEBUDDY.md PRD_V6_AI与动画.md 优化规划.md 快速上手_V5.5.md 项目协作说明.md

# 6. 移动文档到 docs/ 子目录
mv PRD_V6_AI与动画.md docs/prd/
mv 优化规划.md docs/planning/
mv 快速上手_V5.5.md docs/guides/
mv 项目协作说明.md docs/guides/协作说明_巴蒂版.md

# 7. 检查并提交
git status
git add -A
git commit -m "🔀 merge: 巴蒂最新 V6 代码 + 服务端 + WorkBuddy 分析报告"
git push origin main
```

### 第 2 步：启用 Gitee 自动镜像（元一来做）

元一已经在 GitHub 上准备了 `scripts/sync-gitee.sh`，需要在 `.github/workflows/` 下加一个 `sync-to-gitee.yml`，让每次 push 到 main 时自动同步到 Gitee。

需要你在 Gitee 上创建一个 **私人令牌（Personal Access Token）**：
1. 登录 Gitee → 设置 → 私人令牌
2. 生成一个 token，权限勾选 `projects`
3. 把 token 告诉元一（添加到 GitHub Secrets: `GITEE_TOKEN`）

### 第 3 步：WorkBuddy 后续产出都提交到 GitHub

我后续产出的所有文档（PRD、测试用例、分析报告）都直接提交到 GitHub 主仓的 `docs/` 目录。
