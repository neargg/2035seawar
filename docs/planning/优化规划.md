## 二、优化方向全景图

基于竞品调研（炉石传说、影之诗等头部卡牌游戏）和代码分析，识别出 **6大优化方向、22项需求**：

### 方向一：🎮 游戏玩法深度

| # | 需求 | 描述 | 价值 | 复杂度 | 优先级 |
|---|------|------|------|--------|--------|
| 1.1 | **AI 智能化升级** | 当前AI为随机选择动作（`Math.random()`），升级为策略评估AI，支持简单/普通/困难三档难度 | ⭐⭐⭐⭐⭐ | 中 | P0 |
| 1.2 | **新手引导教程** | 首次进入引导玩家完成一局对战（部署→推进→攻击→结束回合），高亮提示操作 | ⭐⭐⭐⭐⭐ | 中 | P0 |
| 1.3 | **卡组编辑器** | 允许玩家自定义牌组（从阵营卡池中选卡组牌），而非固定预设牌组 | ⭐⭐⭐⭐ | 中 | P1 |
| 1.4 | **战役/剧情模式** | 单人PVE关卡，每关有特定条件（如"用3回合内击毁HQ"），通关解锁新卡 | ⭐⭐⭐ | 高 | P2 |
| 1.5 | **天候/事件系统** | 随机天气事件影响战局（如"暴风雨：飞行器本回合不能攻击"） | ⭐⭐ | 中 | P2 |

## 三、优先级建议

### 🔴 P0 - 立即推进（核心体验闭环）

| 需求 | 理由 |
|------|------|
| AI 智能化升级 | 当前随机AI毫无策略性，人机模式体验差，新手无法从中学习 |
| 新手引导教程 | 17种机制 + 135张卡，无引导的新玩家门槛极高 |
| 战斗动画与特效 | 纯文字日志反馈太弱，攻击无视觉反馈严重影响沉浸感 |
| HTTPS + SSL证书 | 安全基线，浏览器已对HTTP标记不安全，WSS必须基于HTTPS |

### 🟡 P1 - 近期推进（留存与体验增强）

| 需求 | 理由 |
|------|------|
| 设置面板 | 音效开关等基础设置是最低要求 |
| 操作确认与撤销 | 降低误操作挫败感 |
| 回合计时器 | 联机防挂机必备 |
| 战绩统计面板 | 玩家需要看到自己的成长数据 |
| 成就系统 | 提供长期目标，增强留存 |
| 卡组编辑器 | 策略深度的核心，自定义是卡牌游戏灵魂 |
| 数据埋点系统 | 没有数据无法做后续决策 |
| 代码模块化 | 为后续迭代打基础 |
| 多游戏入口页 | nearapp.xyz 定位为游戏平台，需预留扩展 |

### 🟢 P2 - 中期推进（社交与内容扩展）

| 需求 | 理由 |
|------|------|
| 排行榜系统 | 需先有账号系统支撑 |
| 每日挑战 | 增加日活抓手 |
| 对局回放 | 高阶玩家需求 |
| 卡牌图鉴 | 内容展示，增强收藏感 |
| 战役/剧情模式 | 单人内容扩展，工作量大 |
| PWA 离线支持 | 提升访问体验 |
| 用户账号系统 | 排行榜/云同步的基础设施 |
| 联机表情/快捷语 | 社交互动基础 |

### ⚪ P3 - 远期探索

| 需求 | 理由 |
|------|------|
| 观战模式 | 需要稳定联机生态后再做 |
| 天候/事件系统 | 增加随机性，需谨慎平衡 |

## 四、待确认问题清单

1. **目标平台**：大海战定位是手机H5游戏、PC网页游戏，还是两者兼顾？是否计划做成App？
2. **团队资源**：目前有几个人可以投入开发？前后端分别谁负责？
3. **商业化方向**：是纯免费游戏还是有内购计划？这会影响账号系统和数据系统的设计
4. **AI难度**：AI智能化升级需要多大的策略深度？是否参考某些固定策略还是需要机器学习？
5. **联机服务器**：当前 WebSocket 服务器（111.229.64.124）是你自己的云服务器吗？后续排行榜/账号系统也部署在这上面？
6. **版本节奏**：期望多久迭代一个版本？是否需要赶上某个时间节点上线？
