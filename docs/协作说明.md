## 版本约定

格式：`V{主版本}.{次版本}.{补丁}` 或 `V{主版本}.{次版本}`

- **主版本**：架构变更（V4 → V5 → V6）
- **次版本**：新功能模块（V6.0 → V6.1 → V6.2）
- **补丁**：bug 修复（V6.0.1）

## 分支策略（建议）

| 分支 | 用途 | 合并到 |
|------|------|--------|
| `main` | 稳定版本，可部署 | - |
| `develop` | 集成开发分支 | `main` |
| `feature/v6.1-xxx` | 单个功能开发 | `develop` |
| `fix/v6.0-tdz-bug` | bug 修复 | `develop` |

## Commit 约定

格式：`<emoji> <type>(<scope>): <subject>`

常用 emoji：
- 🎮 `:video_game:` 新功能/版本
- 🐛 `:bug:` 修复 bug
- 📝 `:memo:` 文档
- 🎨 `:art:` UI/样式
- ⚡ `:zap:` 性能
- 🧪 `:test:` 测试
- 🚀 `:rocket:` 部署
- 🔧 `:wrench:` 配置/脚本

例子：
```
🎮 feat(v6.0): add AI scoring engine
🐛 fix(v6.0): resolve TDZ initialization bug
📝 docs: update PRD v6.0
🚀 deploy: v6.0 to tencent cloud
```

## 任务流转

```
知总拍板 → 巴蒂开发（客户端） + 元一开发（服务器/测试） → 知总实测 → 拍板 release
```

## 测试要求

任何 push 到 `main` 必须：
- ✅ 通过 `tests/ai-engine.test.js`（CI 自动跑）
- ✅ 通过 `tests/v6-e2e.test.js`（CI 自动跑）
- ✅ 知总手动在浏览器验证（巴蒂/元一不能代验）

## 部署流程

1. 巴蒂 push 到 `develop`，CI 自动跑测试
2. 知总 review + 在 develop 分支手动验证
3. 知总 merge `develop` → `main`
4. CI 自动部署到 GitHub Pages（预览）
5. 知总手动 `ssh` 到腾讯云正式部署（用 `deploy/deploy-to-tencent.sh`）
